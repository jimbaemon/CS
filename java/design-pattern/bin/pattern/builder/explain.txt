#빌더 (Builder)
- 하위클래스가 부분객체를 생성하고 디렉터 객체가 조립 절차에 따라 조립하여 복합객체를 생성하는 패턴

##구성요소
1.Builder
 - 객체를 작성하기 위한 인터페이스를 정의한다.
 - 객체의 각 부분을 만들기 위한 추상메서드를 선언한다.
 - 마지막 결과를 얻기위한 메서드를 정의한다.

2.ConcreteBuilder
 - Builder 인터페이스를 구현한다.

3.Director
 - Builder 인터페이스를 사용해서 객체를 생성한다.
 - 각 부분을 조립하는 메서드를 제공한다.

4.Client
 - Director 객체를 사용하여 원하는 기능을 수행한다.

##사용효과
 - 여러 단계와 다양한 절차를 통해서 객체를 만들 수 있다.
 - 복합객체의 내부 구조를 클라이언트로부터 보호할 수 있다.
 - 복합객체 생성 과정을 클래스화하여 복합객체를 동일하게 생성하는 여러 곳의 코드 중복을 막아 준다.

##느낀점
 - 빌더를 중심으로 생성된 객체들이 디렉터를 중심으로 실행되는 듯한 패턴이다?? 뭔가 복잡한데 장점을 잘 모르긴 하곘다.
